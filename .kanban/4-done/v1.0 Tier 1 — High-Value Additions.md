## T1.1 ‚Äî Runtime Error Message Quality

### Current State (Verified)

Error messages currently include contextual variable names in some cases (`"Undefined variable 'foo'."`) but do **not** include the received runtime type. Key deficiencies found across `VirtualMachine.*.cs` and `RuntimeOps.cs`:

| Current Message                          | Problem                                         |
| ---------------------------------------- | ----------------------------------------------- |
| `"Operands must be numbers or strings."` | No mention of what types were actually provided |
| `"Assignment to constant variable."`     | Doesn't name the variable                       |
| `"Division by zero."`                    | Acceptable ‚Äî no type info needed                |
| `"Operands must be numbers or strings."` | Left and right types both unknown to the user   |

### Requirements

Audit all `throw new RuntimeError(...)` calls across:

- `Stash.Bytecode/Runtime/RuntimeOps.cs` ‚Äî arithmetic, comparison, equality dispatch
- `Stash.Bytecode/VM/VirtualMachine.Variables.cs` ‚Äî variable access/assignment
- `Stash.Bytecode/VM/VirtualMachine.Arithmetic.cs` ‚Äî arithmetic fast paths
- `Stash.Bytecode/VM/VirtualMachine.TypeOps.cs` ‚Äî field access, indexing
- `Stash.Stdlib/BuiltIns/*.cs` ‚Äî namespace function errors

For each error, apply these four improvements where applicable:

**1. Include the received type(s)**
The `IVMTyped` protocol provides `TypeName` on protocol-implementing types. For raw strings and `List<StashValue>`, use `"string"` and `"array"`. For `StashValue`, call `StashValue.TypeName(v)` (or equivalent helper ‚Äî verify the actual method name in `StashValue.cs`).

```csharp
// Before
throw new RuntimeError("Operands must be numbers or strings.");

// After
throw new RuntimeError($"Operands must be numbers or strings, got '{leftType}' and '{rightType}'.");
```

**2. Name the variable in assignment/access errors**

```csharp
// Before
throw new RuntimeError("Assignment to constant variable.");

// After
throw new RuntimeError($"Cannot assign to constant '{name}'.");
```

**3. Include stdlib function context**
Stdlib errors should state the function name:

```csharp
// Before (inside arr.get)
throw new RuntimeError("Index out of range.");

// After
throw new RuntimeError($"arr.get: index {index} out of range for array of length {arr.Count}.");
```

**4. Add fix suggestions where the fix is obvious**
Only for errors where the fix is deterministic:

- Access to constant: "Use `let` instead of `const` if mutation is needed."
- Null dereference: "Check for null before accessing '{member}'."
- Type mismatch in arithmetic: "Convert with `conv.toInt()` or `conv.toFloat()` first."

### Scope Boundaries

- Do **not** add error codes to runtime errors in this pass (that's a separate design decision)
- Do **not** change the `RuntimeError` class structure ‚Äî `Span`, `ErrorType`, `Message` stay as-is
- Changes are purely message string improvements ‚Äî no behavior changes
- HTTP/command errors surfacing raw .NET exception text: wrap in a clear prefix (`"http.get: request failed ‚Äî "`) but do not suppress the underlying message

### Tests

Add tests in existing test files that assert on error message content where messages are improving. Each changed message should have at least one corresponding test that verifies the new format includes the type or variable name.

## T1.3 ‚Äî `io.readPassword(prompt)`

### Current State (Verified)

`IoBuiltIns.cs` has 6 functions: `println`, `print`, `eprintln`, `eprint`, `readLine`, `confirm`. No masked-input function exists.

### Function Signature

```stash
let pw = io.readPassword("Enter database password: ")
// Returns: secret
// Prints prompt to stdout without newline
// Does not echo typed characters
// Returns secret type (not string) ‚Äî prevents accidental logging
```

### Semantics

- **Returns `secret` type** ‚Äî identical to how `StashSecret` works throughout the codebase. Prevents the value from appearing in `io.print()`, `log.*()`, interpolation in `"${pw}"`, etc.
- **Non-interactive fallback** ‚Äî if stdin is not a TTY (piped input), reads normally without masking (same as `io.readLine`). This is standard Unix behavior (`sudo` does this too).
- **Cancellation** ‚Äî Ctrl+C / Ctrl+D during input: throw a `RuntimeError` with message `"io.readPassword: input cancelled."`.
- **Empty input** ‚Äî allowed; returns `secret("")`.

### Implementation Notes

**Platform-specific masking:**

- **.NET approach**: `Console.ReadKey(intercept: true)` in a loop, handling `ConsoleKey.Enter` (done), `ConsoleKey.Backspace` (remove last char), and printable characters. Do **not** print `*` by default ‚Äî no echo is the standard.
- **Non-TTY detection**: `Console.IsInputRedirected` ‚Üí if `true`, fall back to `Console.ReadLine()` and wrap in secret.
- **Native AOT compatible**: `Console.ReadKey` is AOT-safe.

**Location:** `Stash.Stdlib/BuiltIns/IoBuiltIns.cs` ‚Äî add as the 7th function, registered as `"readPassword"`.

**Return type:** Wrap the collected string in `new StashSecret(collected)` (verify exact constructor in `Stash.Core/Runtime/Types/StashSecret.cs`).

### Tests

Add to `Stash.Tests/Interpreting/IoBuiltInsTests.cs`:

- Non-TTY (piped) mode returns secret type
- Empty input returns secret
- Prompt is printed (verify via `captureOutput` if applicable)

Note: Interactive masking cannot be tested in unit tests (requires a real TTY). Document this limitation in a test comment.

### LSP / Static Analysis

No changes required. `secret` is an existing type. No new keywords or syntax.

## T1.6 ‚Äî `fs.chown(path, uid, gid)`

### Current State (Verified)

`FsBuiltIns.cs` has 36+ functions. Permission model uses a struct-based approach (`FilePermissions` struct with `owner`/`group`/`others` sub-structs for `read`/`write`/`execute`). No numeric uid/gid ownership change function exists.

### Design Decision

**Ownership change is Unix-only.** There is no meaningful equivalent on Windows (NTFS uses SIDs/ACLs, not uid/gid). The function should:

- Work on Linux and macOS (via P/Invoke `chown`)
- Throw a clear error on Windows: `"fs.chown: file ownership change by uid/gid is not supported on Windows. Use fs.setPermissions() for access control."`

### Function Signature

```stash
fs.chown("/etc/app.conf", 1000, 1000)
// path: string   ‚Äî file or directory path
// uid: int       ‚Äî new owner user ID (-1 to leave unchanged)
// gid: int       ‚Äî new owner group ID (-1 to leave unchanged)
// Returns: null
// Throws on Windows, permission errors, or missing file
```

Using `-1` to mean "don't change" follows the POSIX `chown()` convention.

### Implementation

**.NET does not expose `chown` directly.** Two viable approaches:

**Option A: P/Invoke (preferred)**

```csharp
[DllImport("libc", SetLastError = true)]
static extern int chown(string pathname, uint owner, uint group);
```

- Works on Linux and macOS
- AOT-compatible (P/Invoke is supported in Native AOT)
- Guard with `RuntimeInformation.IsOSPlatform(OSPlatform.Windows)` for the error throw

**Option B: `Mono.Posix.NETStandard` NuGet package**

- Higher-level API (`UnixFileInfo.SetOwner()`)
- Adds a NuGet dependency ‚Äî prefer Option A to avoid bloat

**Implementation location:** `Stash.Stdlib/BuiltIns/FsBuiltIns.cs`

**Error mapping from errno:**

- `EPERM` ‚Üí `"fs.chown: permission denied ‚Äî must be superuser or file owner."`
- `ENOENT` ‚Üí `"fs.chown: file not found: '{path}'."`
- `EINVAL` ‚Üí `"fs.chown: invalid uid {uid} or gid {gid}."`
- Other ‚Üí `"fs.chown: failed with errno {errno}."`

### Tests

Add to `Stash.Tests/Interpreting/FsBuiltInsTests.cs`:

- On Unix: chown to current user's uid/gid succeeds (no privilege required)
- Passing `-1` for uid leaves owner unchanged (verify via `stat`)
- On Windows: throws the platform error message
- Missing file throws appropriate error

Use `[SkipOnPlatform]` or runtime OS checks in tests where platform-specific behavior differs.

## T1.8 ‚Äî `xml` Namespace

### Current State (Verified)

No `XmlBuiltIns.cs` exists. No XML parsing capability anywhere in the Stash stdlib. The only XML in the project is C# internal tooling (Windows Task Scheduler XML generation, project files).

### Data Model

XML cannot be represented as a flat dict like JSON because it has both attributes and child elements. The canonical Stash representation uses a struct hierarchy:

```stash
struct XmlNode {
    tag: string,       // Element name. "#text" for text nodes, "#cdata" for CDATA sections
    attrs: dict,       // Attribute name ‚Üí string value (empty dict if no attributes)
    text: string,      // Direct text content of this element (trimmed, empty if none)
    children: array    // Array of XmlNode structs (child elements, in document order)
}
```

**Design rationale:**

- Struct over dict: consistent with Stash's design philosophy; field names are discoverable via LSP
- `#text` tag for text nodes: follows DOM convention; allows mixed-content documents
- `text` field on element nodes: contains the concatenated direct text content (not recursive) for the common case of `<tag>value</tag>` ‚Üí `node.text == "value"`
- No namespace support in v1 ‚Äî namespace-qualified names are kept as-is (e.g., `"xs:element"`)

**Example:**

```xml
<server host="localhost" port="8080">
    <timeout>30</timeout>
    <tags>
        <tag>web</tag>
        <tag>api</tag>
    </tags>
</server>
```

Maps to:

```stash
XmlNode {
    tag: "server",
    attrs: { "host": "localhost", "port": "8080" },
    text: "",
    children: [
        XmlNode { tag: "timeout", attrs: {}, text: "30", children: [] },
        XmlNode { tag: "tags", attrs: {}, text: "", children: [
            XmlNode { tag: "tag", attrs: {}, text: "web", children: [] },
            XmlNode { tag: "tag", attrs: {}, text: "api", children: [] }
        ]}
    ]
}
```

### Function Signatures

```stash
// Parse XML string ‚Üí XmlNode (root element)
let root = xml.parse(text)

// Parse with options
let root = xml.parse(text, { preserveWhitespace: false })  // default: false

// Serialize XmlNode tree ‚Üí XML string
let text = xml.stringify(root)
let text = xml.stringify(root, { indent: 2, declaration: true })

// Validate well-formedness (does not validate against schema)
let ok = xml.valid(text)   // bool ‚Äî true if parseable, false otherwise

// XPath query ‚Äî returns array of matching XmlNode or string values
let nodes = xml.query(root, "//server/tags/tag")
let values = xml.query(root, "//server/@port")   // Attribute query ‚Üí array of strings
```

### Options Structs

```stash
struct XmlParseOptions {
    preserveWhitespace: bool   // Keep whitespace-only text nodes (default: false)
}

struct XmlStringifyOptions {
    indent: int        // Spaces per indent level (0 = no indentation, default: 2)
    declaration: bool  // Include <?xml version="1.0"?> header (default: true)
    encoding: string   // Encoding attribute in declaration (default: "UTF-8")
}
```

### Implementation

**.NET provides `System.Xml.Linq` (LINQ to XML)** ‚Äî available in all .NET runtimes including Native AOT (as of .NET 7+). Use `XDocument.Parse()` and `XElement` for parse/stringify. Use `XElement.XPathSelectElements()` (from `System.Xml.XPath`) for `xml.query`.

**Verify AOT compatibility** of `System.Xml.Linq` and `System.Xml.XPath` before committing. Add trim annotations if needed. These are well-known to work with Native AOT.

**Location:** `Stash.Stdlib/BuiltIns/XmlBuiltIns.cs` (new file)

**Struct registration:** `XmlNode`, `XmlParseOptions`, `XmlStringifyOptions` must be registered in the stdlib metadata (follow the pattern from `CsvBuiltIns.cs` or `HttpBuiltIns.cs` for struct registration).

**Error handling:**

- Parse error ‚Üí `RuntimeError("xml.parse: invalid XML ‚Äî {position}: {message}.")`
- XPath error ‚Üí `RuntimeError("xml.query: invalid XPath expression ‚Äî {message}.")`

### Tests

New file: `Stash.Tests/Interpreting/XmlBuiltInsTests.cs` ‚Äî minimum 25 tests covering:

| Group           | Scenarios                                                                                                   |
| --------------- | ----------------------------------------------------------------------------------------------------------- |
| `xml.parse`     | Simple element, element with attrs, nested elements, text content, CDATA, comments (ignored), empty element |
| `xml.stringify` | Round-trip parse‚Üístringify‚Üíparse, indentation, XML declaration                                              |
| `xml.valid`     | Valid XML ‚Üí true, malformed XML ‚Üí false, empty string ‚Üí false                                               |
| `xml.query`     | Element selection, attribute selection, nested path, no-match returns empty array                           |
| Edge cases      | Empty document, unicode content, special chars in attrs, whitespace handling                                |

### Documentation

Add `xml` section to `docs/Stash ‚Äî Standard Library Reference.md` following the existing namespace doc format. Include the `XmlNode` struct definition and all 4 function signatures with examples.

### LSP / Static Analysis

- LSP hover should surface `XmlNode` struct fields (automatic via struct registration)
- No new static analysis rules needed in v1

## T1.9 ‚Äî TAP Framework ‚Äî `test.only()` + `assert.deepEqual()`

### Current State (Verified)

### T1.9a ‚Äî `test.only(name, fn)`

#### Semantics

When one or more `test.only()` calls exist in a file:

- Only those tests run; all `test.it()` calls are silently skipped
- `test.skip()` tests remain skipped (as normal)
- `test.only()` inside a `test.describe()` only affects that describe block ‚Äî tests outside are still skipped globally

This is the standard behavior of Jest's `test.only` / `it.only`. The implementation tracks an "exclusive mode" flag.

#### Implementation

**In `TestBuiltIns.cs`:**

- Register `"only"` alongside `"it"` ‚Äî similar signature
- Set a global `hasExclusiveTests` flag on first `test.only()` call
- Mark the test entry with `isExclusive: true`

**In the test runner (TapReporter / execution loop):**

- After collecting all tests: if `hasExclusiveTests` is true, skip all non-exclusive tests
- Skip message for non-exclusive tests: `"# skip (test.only active)"`

**`describe` scoping:** If `test.only()` appears inside a `describe()`, the exclusive flag is scoped to that describe block. Tests outside the describe are still skipped. If `test.only()` appears at the top level, all tests not marked `only` are skipped globally.

#### Tests

Add to `Stash.Tests/Interpreting/TestBuiltInsTests.cs`:

- `test.only` runs only the exclusive test, not `test.it` peers
- Multiple `test.only` calls all run
- `test.only` inside `describe` skips sibling `it` calls but outer `it` calls too
- Without any `test.only`, `test.it` runs normally (regression)

### T1.9b ‚Äî `assert.deepEqual(actual, expected)`

#### Semantics

Recursive structural equality check:

- **Primitives** (`int`, `float`, `bool`, `string`, `null`): strict equality (same as `assert.equal`)
- **Arrays**: same length, each element recursively deepEqual
- **Dicts**: same keys, each value recursively deepEqual
- **Structs**: same struct type AND same field values recursively deepEqual (use `StashInstance` type name for struct type check)
- **Other types** (`secret`, `byte`, etc.): strict equality

**Failure message format:**

```
AssertionError: deepEqual failed
  Path: .children[1].attrs["host"]
  Expected: "localhost"
  Actual:   "127.0.0.1"
```

The `Path` traces how the difference was found. Use dot notation for fields, bracket notation for array indices and dict keys.

#### Implementation

**In `AssertBuiltins.cs`:**

- New function `"deepEqual"` registered in the `assert` namespace
- Recursive comparison function `DeepEquals(StashValue actual, StashValue expected, string path)` returning `(bool equal, string? failPath, string? expectedStr, string? actualStr)`
- On mismatch: throw `AssertionError` with the formatted message above

**Cycle detection:** Track visited reference pairs to prevent infinite recursion on circular structures. A `HashSet<(int, int)>` of identity hash pairs (by reference) is sufficient.

#### Tests

Add to `Stash.Tests/Interpreting/TestBuiltInsTests.cs` or a new `AssertBuiltInsTests.cs`:

- Primitive equality (int, float, bool, string, null)
- Nested arrays
- Nested dicts
- Struct equality (same struct type, same fields)
- Mismatch in array element ‚Üí correct path in error message
- Mismatch in nested dict key ‚Üí correct path
- Different array lengths fail with path
- Different struct types fail

### T1.9c ‚Äî `assert.closeTo(actual, expected, delta)`

#### Semantics

```stash
assert.closeTo(3.14159, math.pi, 0.001)
// Passes if |actual - expected| <= delta
```

- `delta` must be ‚â• 0 (throw if negative: `"assert.closeTo: delta must be non-negative."`)
- Both `actual` and `expected` must be numeric (int or float)
- Failure message: `"Expected {actual} to be within {delta} of {expected} (difference: {|actual-expected|})."`

## Decision Log

| Date       | Decision                                                    | Rationale                                                                                 |
| ---------- | ----------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| 2026-04-26 | `io.readPassword` returns `secret` type                     | Prevents accidental logging; `secret` already exists in the type system                   |
| 2026-04-26 | `xml.parse` returns `XmlNode` struct, not dict              | Struct-over-dict preference; LSP discoverability; struct fields are self-documenting      |
| 2026-04-26 | `xml` namespace uses `System.Xml.Linq` (LINQ to XML)        | Ships in BCL, no extra NuGet dependency, full XPath support, AOT-compatible               |
| 2026-04-26 | `fs.chown` uses P/Invoke over Mono.Posix NuGet              | Avoids adding a dependency; P/Invoke is AOT-safe; Mono.Posix is overkill for one syscall  |
| 2026-04-26 | `fs.chown` throws on Windows (no graceful no-op)            | Silent no-op would mask portability bugs; clear error helps scripts fail fast on wrong OS |
| 2026-04-26 | `test.only` scoped to describe block when called inside one | Matches Jest behavior; reduces surprise; allows focused debugging at any level            |
| 2026-04-26 | `assert.deepEqual` includes failure path in error message   | Without path info, debugging nested structure mismatches is extremely painful             |
| 2026-04-26 | Actual namespace count is 35 (not 30 or 31)                 | Verified by listing Stash.Stdlib/BuiltIns/ ‚Äî 35 registered namespaces                     |
