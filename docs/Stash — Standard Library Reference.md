## How to Read This Reference

Every namespace section follows the same shape: a brief purpose statement, a capability requirement,
a list of types/constants the namespace registers, a function summary table, and per-function details.

### Signature notation

Function signatures use Stash-facing types — the C# implementation type is not exposed.

```
namespace.function(required: type, optional?: type, ...rest) -> returnType
```

- `name: type` — required parameter and its Stash type label.
- `name?: type` — optional parameter; calls may omit it.
- `...rest` — variadic tail; remaining arguments are bundled into an array.
- `-> type` — the Stash return type. `null` means the function returns no useful value.

### Type labels

| Label                  | Meaning                                                       |
| ---------------------- | ------------------------------------------------------------- |
| `int`                  | 64-bit signed integer                                         |
| `float`                | 64-bit IEEE-754 double                                        |
| `number`               | Accepts either `int` or `float`                               |
| `string`               | UTF-16 string                                                 |
| `bool`                 | Boolean                                                       |
| `byte`                 | Unsigned 8-bit integer (0–255)                                |
| `buffer`               | Byte array                                                    |
| `array`                | Heterogeneous array of `any`                                  |
| `array<T>` / `T[]`     | Typed array                                                   |
| `dict`                 | String-keyed dictionary                                       |
| `function`             | A callable value                                              |
| `Future<T>`            | Pending result returned by an `async` function or `task.run`  |
| `secret`               | A `secret(...)`-wrapped value; auto-redacts on stringify      |
| Named struct / enum    | A built-in struct or enum from the same namespace             |
| `any`                  | Any Stash value                                               |

### Errors and throws

Built-in error types are documented in [Global Functions and Types](#global-functions-and-types).
Each function lists the error types it may throw under **Throws**. Typed `catch` clauses match
the listed types — see the Language Specification's error-handling section.

### Capability gates

Some namespaces require capabilities to be enabled. Embedded hosts can omit capabilities to disable
the matching namespaces entirely. The capability requirement is shown at the top of each namespace
section as **Capability:** `Network` / `FileSystem` / `Process` / `Environment` / `Shell` /
`Scheduler`, or _none_ when the namespace is always available.

### Deprecation

Deprecated members are marked _(deprecated — use `replacement`)_ and surface through the static
analysis rule `SA0830`. A consolidated list lives in [Appendix A](#appendix-a--deprecated-members).

## Global Functions and Types

Names in this section are reachable without a namespace qualifier — they live directly in the

### Built-in Error Types

Stash defines a set of built-in named error types. These types are globally visible — no import
required. They can be thrown with struct-literal syntax and caught by typed `catch` clauses.

| Type | Fields | When thrown |
| ---- | ------ | ----------- |
| `AliasError` | `message: string`, `aliasName: string?`, `detail: string?` | Alias name is invalid or conflicts with an existing definition. |
| `CancellationError` | `message: string` | External cancellation (Ctrl-C or programmatic cancellation). |
| `CommandError` | `message: string`, `exitCode: int`, `stderr: string`, `stdout: string`, `command: string` | Strict command ($!(...)) exited with a non-zero exit code. |
| `IOError` | `message: string` | File or network I/O failure. |
| `IndexError` | `message: string` | Array or string index is out of bounds. |
| `LockError` | `message: string`, `path: string` | File-lock acquisition failed. |
| `NotSupportedError` | `message: string` | Operation is not supported on this platform or for this value. |
| `ParseError` | `message: string` | Parsing failed (JSON, INI, TOML, CSV, number conversion, etc.). |
| `StateError` | `message: string` | Object is in an invalid state for the requested operation. |
| `TimeoutError` | `message: string` | Operation timed out (timeout block or stdlib timeout parameter). |
| `TypeError` | `message: string` | Argument or operand has the wrong type. |
| `ValueError` | `message: string` | Value is invalid for the operation (out-of-range, empty, or malformed). |

All caught errors expose `.message: string`, `.type: string`, and `.stack: array<string>` in
addition to any type-specific fields shown above.

#### `alias.exec(name: string, args: array) -> int`

**Returns:** `int` — Exit code (int)

#### `alias.expand(name: string, args: array) -> string`

**Parameters:**

**Returns:** `string` — Expanded string

**Throws:**

- `AliasError` — if the alias is not defined
- `TypeError` — if any element of args is not a string

#### `alias.save(...rest: ...any) -> string`

#### `alias.load(...rest: ...any) -> int`

**Parameters:**

- `rest`: `...any`

#### `archive.unzip(archivePath: string, outputDir: string, ...options: any) -> array`

Extracts a ZIP archive to a directory.

**Parameters:**

- `options`: `any` — Optional ArchiveOptions struct with overwrite, preservePaths, filter (glob pattern)

- `ValueError` — if an entry would extract outside the target directory or options fields have invalid values

#### `archive.tar(outputPath: string, inputPaths: any, ...options: any) -> string`

Creates a TAR archive from one or more files or directories.

#### `fs.watch(path: any, callback: any, ...options: any) -> Watcher`

Watches a file or directory for changes and invokes a callback for each event. Returns a Watcher handle that can be passed to fs.unwatch() to stop watching.

The callback receives a WatchEvent struct with type (a WatchEventType enum: Created, Modified, Deleted, Renamed), path (absolute), and oldPath (for renames only). Callbacks execute in an isolated forked context. Value-type variables from the parent scope are snapshotted; mutations to reference types (dicts, struct instances) are visible in both directions. Events are debounced by default (100ms window) to suppress duplicate OS notifications. Set debounce to 0 in WatchOptions to receive every raw event.

**Parameters:**

- `path`: `any` — File or directory path to watch.
- `callback`: `any` — Function receiving a WatchEvent on each change.
- `options`: `any` — Optional WatchOptions struct: recursive (bool), filter (string glob), bufferSize (int bytes), debounce (int ms).

**Returns:** `Watcher`

**Throws:**

- `IOError` — if the path does not exist or the watcher cannot be started
- `TypeError` — if `path` or `callback` has the wrong type

#### `fs.unwatch(watcher: any) -> null`

Stops a file watcher previously created by fs.watch(). Disposes the underlying OS watcher and removes it from tracking. Calling fs.unwatch() on an already-stopped watcher is a no-op.

**Parameters:**

**Returns:** `null`

**Throws:**

- `TypeError` — if `watcher` is not a Watcher handle
