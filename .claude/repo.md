## Architecture Decisions (enduring)

- **Bytecode VM over tree-walk interpreter** — the VM is the primary execution engine. The tree-walk interpreter (`Stash.Interpreter`) still exists for legacy/tests but new features target the bytecode path.
- **Native AOT for CLI only** — `Stash.Cli`, `Stash.Check`, `Stash.Format` are AOT. `Stash.Lsp`, `Stash.Dap`, `Stash.Registry` must NEVER be AOT (OmniSharp reflection requirement).
- **`IStdlibProvider` injection** — stdlib built-ins are injected into the VM, not referenced directly. Keeps `Stash.Bytecode` from depending on `Stash.Stdlib`.
- **Dispatch loop size limit** — the switch dispatch in `VirtualMachine.Dispatch.cs` is near .NET AOT's optimization threshold. Adding opcodes risks 10–20% perf regression. Always check L1-icache-load-misses before/after.
- **DiagnosticDescriptors are canonical** — every static analysis diagnostic MUST go through `DiagnosticDescriptors.cs`. Never hand-construct `SemanticDiagnostic`.
- **Visitor completeness** — 6 visitors implement `IExprVisitor<T>` / `IStmtVisitor<T>`. Every new AST node must be added to all 6.

## Gotchas & Surprises

- **`Stash.Tests/` maps to Stash.Interpreter architecture**, not Stash.Bytecode — many test helpers use the tree-walk interpreter even though the bytecode VM is primary. Bytecode-specific tests are in `Stash.Tests/Bytecode/`.
- **`kanban/` is `.kanban/`** — the kanban directory has a leading dot. Some older docs reference `kanban/` without the dot.
- **`build.stash` is the build script** — prefer it over raw `dotnet publish` for release builds. It has binary-size guards for AOT binaries.
- **OmniSharp packages are pinned to v0.19.9** — do not upgrade. Later versions have breaking changes to the handler registration API.
